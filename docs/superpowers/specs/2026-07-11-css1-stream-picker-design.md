# CSS1 播放源解析速度設計

## 根因

CSS1 已能保留多站播放線，但播放階段仍逐條解析，總等待時間等於各網站等待時間相加。runtime 在解析開始前已切換到播放畫面，因此解析尚未完成時只會看到黑底；所有播放線完成後，既有來源彈窗才會出現。

## 修正

- CSS1 播放線使用 task group 並行解析，每條線獨立使用自己的來源設定；結果依原始索引與畫質 priority 穩定排序。
- 保留現有來源彈窗、記住來源、單一來源自動播放與 AVPlayer 備援行為。

## 驗證

- 兩條各延遲 150ms 的 CSS1 播放線須在 260ms 內完成解析並保留兩個候選。
- 完整 `TVShellChecks` 通過。
